# 皖工购物网系统用户故事文档

## 1. 项目简介

皖工购物网系统旨在为用户提供一个基础但完整的线上购物环境，使买家能够浏览与购买商品，商家能够发布并管理自身的商品，而系统管理员负责维持平台的正常运作与内容健康。本系统强调易用性、可维护性以及基础电商流程的完整实现，适合作为课程项目的最小可行产品（MVP）。

---

## 2. 第一部分：关键用户角色

| 用户角色  | 角色描述                       |
| ----- | -------------------------- |
| 买家    | 在购物网浏览商品并完成购买的使用者。          |
| 商家    | 在平台发布、更新、管理商品的店铺经营者。       |
| 系统管理员 | 负责审核内容、维护系统稳定、处理违规信息的管理人员。 |

---

## 3. 第二部分：用户故事清单（按角色分组）

### 买家

1. 作为一个买家，我想要浏览商品列表，以便快速了解可购买的商品。
2. 作为一个买家，我想要通过关键词搜索商品，以便更快找到我需要的商品。
3. 作为一个买家，我想要查看商品详情，以便确认商品信息是否符合我的需求。
4. 作为一个买家，我想要将商品加入购物车，以便稍后集中下单购买。
5. 作为一个买家，我想要提交订单并完成支付，以便获得需要的商品。

### 商家

6. 作为一个商家，我想要添加新的商品信息，以便能够向买家展示并销售商品。
7. 作为一个商家，我想要编辑商品库存与价格，以便保持商品信息准确并减少交易纠纷。

### 系统管理员

8. 作为一个系统管理员，我想要下架违规商品，以便保证平台内容的合法性与安全性。
9. 作为一个系统管理员，我想要查看系统基本运行信息，以便及时发现潜在问题并维护平台稳定。
10. 作为一个系统管理员，我想要审核待上架的商品，以便确保商品符合平台规范。

## 4. 第三部分：用户故事与验收标准

| 用户故事             | 验收标准                                                           |
| ---------------- | -------------------------------------------------------------- |
| **1. 买家浏览商品列表**  | ① 买家进入商品列表页面时，应显示所有在售商品名称、价格与缩略图。<br>② 商品数量超过一屏时，系统应支持分页或滚动浏览。 |
| **2. 买家搜索商品**    | ① 当输入存在的关键词时，应显示所有匹配的商品列表。<br>② 当关键词无匹配结果时，应显示"未找到相关商品"的提示。    |
| **3. 买家查看商品详情**  | ① 点击商品后应进入详情页，显示名称、价格、库存、图片与描述。<br>② 若商品已下架，应提示"该商品暂时不可购买"。    |
| **4. 买家加入购物车**   | ① 用户点击"加入购物车"后，应出现成功提示。<br>② 购物车应显示该商品及数量。                     |
| **5. 买家提交订单与支付** | ① 当购物车不为空时，买家可进行下单操作。<br>② 下单成功后系统应显示订单编号、金额与商品列表。             |
| **6. 商家添加商品**    | ① 商家填写名称、价格、库存后可成功创建商品。<br>② 新增商品应提交审核，等待管理员批准后上架。               |
| **7. 商家修改库存与价格** | ① 修改后的库存或价格应即时生效并对买家可见。<br>② 若库存变为 0，应在前端显示"缺货"。               |
| **8. 管理员下架违规商品** | ① 点击"下架"后，该商品不再对买家可见。<br>② 后台界面中该商品需显示"已下架"。                   |
| **9. 管理员查看系统状态** | ① 后台页面应显示订单数量、用户数量等基本运营数据。<br>② 当关键指标出现异常时，管理员应看到警示信息。         |
| **10. 管理员审核商品**  | ① 管理员可以在后台查看待审核的商品列表。<br>② 管理员可以批准商品上架，使其对买家可见。             |

## 5. 第四部分：MoSCoW 优先级排序与理由说明

### MoSCoW 优先级表

| 优先级                 | 用户故事编号           | 描述                            |
| ------------------- | ---------------- | ----------------------------- |
| **M – Must-have**   | 1, 2, 3, 4, 5, 6 | 浏览、搜索、查看详情、购物车、下单、商家商品发布等核心能力 |
| **S – Should-have** | 7, 8, 9, 10      | 商家库存与价格管理、管理员下架违规商品、系统监控功能、商品审核功能 |
| **C – Could-have**  | -                | -                             |
| **W – Won't-have**  | -                | -                             |

### Must-have 优先级理由

* **浏览商品（故事 1）**是整个购物网的基础能力，买家无法浏览商品即无法进行任何购物行为，因此必须实现。
* **下单支付（故事 5）**是促成交易的关键步骤，若无法下单，则购物网无法发挥"购买"功能，是最核心的业务场景，因此优先级最高。
* **商品管理（故事 6）**是商家运营的基础，必须实现才能支持商家功能。
* **违规商品下架（故事 8）**是维护平台健康生态的必要功能，必须实现。
* **系统监控（故事 9）**是管理员维护平台正常运行的必要功能，必须实现。
